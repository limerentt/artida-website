# АРТИДА — artida.by

Корпоративный сайт белорусского производителя бесконтактных технологий и электроники для систем безопасности и контроля доступа.

**Live:** https://limerentt.github.io/artida-website/

---

## О проекте

АРТИДА — компания из Минска, которая разрабатывает и производит:
- **МАГИЯ-3** — бесконтактная кнопка выхода с ИК-сенсором (флагман)
- **КТМ-УД** — считыватель карт для систем контроля доступа
- **КЕ-01** — электронный контроллер
- LoRaWAN телеметрия, контрактное производство печатных плат

Сайт — это визитка + каталог продукции с упором на визуальную презентацию флагманского продукта МАГИЯ-3 через scroll-driven 3D-анимацию.

## Hero-секция (index.html) — ключевая часть

Главная страница содержит **scroll-driven анимацию** разборки/сборки продукта МАГИЯ-3.

### Архитектура

```
.canvas-container          (height: 320vh — длинный скролл-контейнер)
└── .canvas-sticky         (position: sticky; height: 100vh — прилипает)
    ├── .side-panel--left  (grid col 1 — спецификации слева)
    ├── .canvas-center     (grid col 2 — flex column)
    │   ├── .hero-intro    (заголовок "МАГИЯ-3" + CTA — фейдится при скролле)
    │   ├── .canvas-wrap   (flex:1 — оборачивает canvas)
    │   │   └── canvas#productCanvas
    │   └── .hero-scroll-hint
    ├── .side-panel--right (grid col 3 — спецификации справа)
    └── .scroll-progress   (4 точки-индикатора фазы)
```

### Как работает scroll-анимация

1. **242 WebP-фрейма** (1928×1072, прозрачный фон) загружаются в `ProductCanvas`
2. GSAP `ScrollTrigger` привязан к `.canvas-container` (320vh)
3. При скролле `progress` (0→1) транслируется в номер фрейма: `frame = progress × 242`
4. Canvas перерисовывается на каждом шаге с `drawImage` (contain-fit)

### Scroll-фазы

| Фаза | Прогресс | Что видно |
|------|----------|-----------|
| Intro | 0–15% | Заголовок "МАГИЯ-3" + CTA кнопка, потом фейдятся |
| Phase 1 | 15–40% | ИК-сенсор, СКУД (боковые панели) |
| Phase 2 | 40–65% | 1М+ циклов, 12–24V |
| Phase 3 | 65–100% | IP65, LED-индикация |

Боковые панели `.side-panel` содержат блоки `.spec-block[data-phase]`, которые появляются/исчезают с классом `.visible` в зависимости от текущей фазы.

### ProductCanvas (класс)

- **DPI-aware**: умножает размер canvas на `devicePixelRatio`
- **Container-sizing**: подстраивается под размер родителя `.canvas-wrap`, не под окно
- **Fallback**: если фреймы не загрузились — показывает `magiya-hero.jpg`
- **Contain-fit**: масштабирует изображение с сохранением пропорций (`CANVAS_IMAGE_SCALE: 1.0`)

## Как были подготовлены фреймы

Источник: два видео из **Higgsfield AI** (генерация 3D-анимации по фото):
- `assembling_blue.mp4` — сборка продукта (121 кадров)
- `disassembling_blue.mp4` — разборка продукта (121 кадров)

Видео имели однородный голубой фон RGB(134, 230, 247).

### Обработка (Python + Pillow + SciPy)

1. Извлечение кадров из видео через `ffmpeg`
2. **Chroma key**: удаление голубого фона с мягким feathering
   - Hard cutoff: ±10 от целевого цвета по каждому каналу
   - Soft edge: ±10–25 — линейное затухание альфы
   - 1px binary erosion (SciPy) для удаления артефактов на границах
3. Конвертация в WebP (lossy, quality 90)
4. Итого: 242 фрейма (121 disassembling + 121 assembling), ~80 KB каждый

**Важно:** AI-модели для удаления фона (u2net, ISNet, BiRefNet) давали плохие результаты с артефактами. Простой chroma key оказался идеальным решением для однородного фона.

## Запуск локально

```bash
# Любой статический сервер — нужен для загрузки фреймов (CORS)
python3 -m http.server 8080
# или
npx serve .
# или
php -S localhost:8080
```

Открыть http://localhost:8080.

**Нельзя** просто открыть `index.html` как файл — canvas-фреймы не загрузятся из-за политики Same-Origin.

## Адаптивность

Breakpoint: `@media (max-width: 900px)` — переключается на одноколоночный layout:
- Боковые панели `.side-panel` скрываются
- Canvas занимает всю ширину
- `.scroll-progress` (точки-индикаторы) скрывается
- Бургер-меню вместо десктопной навигации
