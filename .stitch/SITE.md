## 1. Назначение сайта

Витрина + воронка в контакт. Не интернет-магазин.

**Задачи (от собственника, по приоритету):**
1. Показать что АРТИДА — серьёзный производитель, а не перекупщик
2. Привлекать заказчиков на контрактное производство электроники
3. Привлечь новых дилеров / партнёров
4. Повысить узнаваемость бренда

**CTA:** позвонить / написать на почту

---

## 2. Карта сайта (sitemap)

```
/                     → Главная
/production           → Контрактное производство ★ ключевая
/catalog              → Каталог продукции
/catalog/[slug]       → Страница продукта (×7)

/news/[slug]          → Страница новости
/contacts             → Контакты
/admin                → CMS-панель (закрыта)
```

### 3.3 Каталог продукции `/catalog`

**Цель:** витрина компетенций. Каждый продукт доказывает: АРТИДА реально производит.

**Layout:** сетка карточек ProductCard (3 колонки desktop, 2 tablet, 1 mobile).

**Фильтрация:** по категориям (Контроль доступа / Охранная сигнализация) — если нужно. При 7 продуктах возможно достаточно простого списка.

**Карточка ProductCard:**
- Фото (4:3)
- Название (h3)
- Краткое описание (1 строка)
- Цена РРЦ (BYN)
- Кнопка «Подробнее»

- Список дилеров рядом с картой (или под ней на мобильном).

**Данные на каждого дилера:**

- Адрес
- Телефон (кликабельный)
- Email
- Сайт

### 3.8 Документация и сертификаты `/docs`

**Цель:** загрузка файлов. Даташиты, сертификаты, техпаспорта.

**Layout:** список/сетка документов, группировка по типу.

**Группы:**
- Руководства по эксплуатации
- Технические паспорта
- Декларации соответствия
- Сертификаты
- Прочие документы (письма БелГИСС и т.д.)

**Каждый документ:**
- Иконка типа файла (PDF, JPG)
- Название
- Тип документа
- Размер файла
- Кнопка «Скачать»

**Контент:** SITE_CONTENT.md §5 (Документы), §6 (Сертификаты).

### 3.9 Новости / блог `/news`

**Цель:** SEO + демонстрация активности компании.

**Layout:** список новостей (карточки). Сортировка: от новых к старым.

**Карточка новости:**
- Обложка (если есть)
- Дата публикации
- Заголовок
- Excerpt (2-3 строки)

**Страница новости `/news/[slug]`:**
- Заголовок (h1)
- Дата
- Контент (markdown → HTML)
- Кнопка «Назад к новостям»

**Контент:** SITE_CONTENT.md §10 (Новости — 7 шт. с 2018 года). Нужны новые новости для запуска.
