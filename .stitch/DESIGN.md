# DESIGN.md — Дизайн-система АРТИДА v2

> Все решения основаны на OWNER_VISION.md. Ключевой запрос собственника: «простой но представительный дизайн», «сильная подача предприятия». Не красиво, не модно — СИЛЬНО.

---

## 1. ДИЗАЙН-ФИЛОСОФИЯ

### Принцип: Промышленный минимализм

Собственник хочет **«сильно» подать предприятие** — как серьёзный промышленный игрок. Текущий сайт: «слабый дизайн, не хватает презентабельности предприятия и услуг».

**Три столпа дизайна:**

1. **МОЩЬ, не красота.** Каждый элемент доказывает: это реальный производитель с итальянским SMD-оборудованием, а не перекупщик. Крупные цифры, фото производства, инженерные схемы.

2. **ПРОСТОТА, не скудность.** «Простой но представительный» — чистые линии, много воздуха, без визуального шума. Руководитель направления (ЦА) ценит скорость получения информации — никакого маркетингового тумана.

3. **ДОВЕРИЕ через доказательства.** Сертификаты, даташиты, схемы подключения, реальные фото. Не лозунги «мы лучшие» — конкретные артефакты, подтверждающие компетентность.

### Антипаттерны (чего НЕ делаем)

- ❌ Модные градиенты, glassmorphism, неоморфизм — это не tech-стартап
- ❌ Стоковые фото улыбающихся людей — это B2B-производство
- ❌ Тяжёлые анимации — скорость = приоритет #1 собственника
- ❌ Canvas-анимация из v1 (242 фрейма × 80KB = 19MB) — прямой конфликт с требованием скорости
- ❌ Перегруженные страницы — «простой дизайн» из уст собственника
- ❌ Маркетинговый пафос — тон «деловой, но дружелюбный»

- **Контраст:** весь текст проходит WCAG AA (≥ 4.5:1 для body, ≥ 3:1 для large text). Charcoal на White = 12.9:1 ✓, White на Deep Navy = 14.4:1 ✓.

## 3. ТИПОГРАФИКА

### Шрифт: IBM Plex Sans

**Почему:** промышленный характер (создан для IBM), отличная кириллица, геометричный но читаемый, есть все нужные начертания, open-source.

### Шкала

```
--font-size-xs:   0.75rem   (12px)   — мета, сноски
--font-size-sm:   0.875rem  (14px)   — подписи, бейджи
--font-size-base: 1rem      (16px)   — body text
--font-size-lg:   1.125rem  (18px)   — lead-текст, интро
--font-size-xl:   1.25rem   (20px)   — h4
--font-size-2xl:  1.5rem    (24px)   — h3
--font-size-3xl:  1.875rem  (30px)   — h2
--font-size-4xl:  2.25rem   (36px)   — h1
--font-size-5xl:  3rem      (48px)   — hero-заголовок (desktop)
--font-size-6xl:  3.75rem   (60px)   — hero-число (статистика)
```

### Стили текста

| Элемент | Size | Weight | Line-height | Letter-spacing | Пример |
|---------|------|--------|-------------|----------------|--------|
| Hero heading | 5xl→4xl (mob) | 700 | 1.05 | -0.02em | «Производство электроники» |
| H1 | 4xl→3xl (mob) | 700 | 1.1 | -0.02em | Заголовок страницы |
| H2 | 3xl→2xl (mob) | 700 | 1.15 | -0.01em | Заголовок секции |
| H3 | 2xl→xl (mob) | 600 | 1.2 | 0 | Заголовок блока |
| H4 | xl→lg (mob) | 600 | 1.3 | 0 | Подзаголовок |
| Body | base | 400 | 1.6 | 0 | Основной текст |
| Body Large | lg | 400 | 1.6 | 0 | Интро, lead-параграфы |
| Small | sm | 400 | 1.5 | 0 | Подписи, мета |
| Caption | xs | 500 | 1.4 | 0.02em | Сноски, бейджи |
| Button | sm | 600 | 1 | 0.02em | Текст кнопок |

### Правила типографики

- **Self-hosted** шрифт (public/fonts/) — без зависимости от Google Fonts, быстрее TTFB.
- **font-display: swap** — текст виден сразу, шрифт подгружается.
- **Веса:** подключать только 400, 500, 600, 700 (не все 9 начертаний).
- **Максимальная ширина строки:** 72ch для body text — для комфортного чтения.
- **Кириллица + латиница** — оба subset'а подключены для i18n.

## 6. ИЗОБРАЖЕНИЯ И МЕДИА

### Фото продукции

- **Формат:** WebP (fallback: JPEG). Генерация через next/image.
- **Фон:** нейтральный белый или Light Frost — единый стиль для всего каталога.
- **Соотношение сторон:** 4:3 для карточек каталога, свободное для страницы продукта.
- **Макс. вес:** 200KB на фото (после оптимизации next/image).
- **Несколько ракурсов:** галерея на странице продукта (собственник запросил).

### Фото производства

- Реальные фото цеха, оборудования, процессов (есть 4 шт. в 240×180, нужны оригиналы).
- Используются на: Главная (hero/секция), О компании, Контрактное производство.
- **Не использовать:** стоковые фото. Только реальное производство АРТИДА.

### Схемы подключения

- Чистые инженерные схемы (SVG предпочтительно, PNG fallback).
- Цвета: brand для активных линий, text-secondary для нейтральных элементов.
- Кликабельны → открытие в полном размере.

### PDF-документы

- Не встраивать в страницу — только скачивание по клику.
- Иконка файла + название + размер + кнопка «Скачать».
- Lazy loading — загрузка ссылки, не контента.

### Иконки

- **Библиотека:** Lucide Icons — чистые, геометричные, в духе промышленного минимализма.
- **Размеры:** 16px (inline), 20px (компоненты), 24px (навигация), 32px (features).
- **Цвет:** наследуют currentColor.
- **Stroke-width:** 1.5px (тоньше дефолтных 2px для элегантности).

## 7. АНИМАЦИИ И ПЕРЕХОДЫ

### Принцип: функциональные, не декоративные

Собственник требует скорость загрузки (#1) + простой дизайн. Анимации — минимальные, только для улучшения UX.

### Допустимые анимации

| Элемент | Тип | Duration | Easing |
|---------|-----|----------|--------|
| Кнопки hover/active | цвет, тень | 150ms | ease |
| Карточки hover | тень, translate | 200ms | ease |
| Навигация появление | fade + translateY | 200ms | ease-out |
| Мобильное меню | slide-in from right | 300ms | ease-out |
| Секции при скролле | fade-in + translateY(20px) | 400ms | ease-out |
| Аккордеон | max-height | 200ms | ease |
| Модальные окна | fade + scale(0.95→1) | 200ms | ease-out |

### Запрещённые анимации

- ❌ Canvas-анимация (19MB из v1)
- ❌ Параллакс-скроллинг (тяжёлый, конфликт со скоростью)
- ❌ Автопроигрывающееся видео в hero
- ❌ Бесконечные loop-анимации
- ❌ Анимации длиннее 500ms

### Реализация

- **Framer Motion** для скролл-анимаций (секции) — LazyMotion + domAnimation для минимального бандла.
- **CSS transitions** для hover/focus/active — не нужен JS.
- **prefers-reduced-motion** — полностью отключать анимации для accessibility.

## 9. ACCESSIBILITY (A11Y)

- **WCAG 2.1 AA** — минимальный стандарт.
- **Контраст:** все цветовые пары проверены (см. раздел 2).
- **Focus:** видимый ring на всех интерактивных элементах (outline-brand, offset 2px).
- **Alt-тексты:** на всех изображениях (продукция, схемы, фото производства).
- **Semantic HTML:** header, nav, main, article, section, footer, aside.
- **Skip-to-content:** ссылка для скрин-ридеров.
- **Клавиатурная навигация:** Tab-order логичный, все CTA доступны.
- **prefers-reduced-motion:** отключение анимаций.
- **prefers-color-scheme:** пока нет тёмной темы (не запрашивалось), но готовность через CSS-переменные.

## 10. АДАПТИВНЫЙ ДИЗАЙН

### Стратегия: Mobile-First

Базовые стили → от мобильного. Расширяем через `@media (min-width: ...)`.

### Ключевые адаптации

| Компонент | Mobile (<768px) | Tablet (768-1024px) | Desktop (>1024px) |
|-----------|-----------------|---------------------|-------------------|
| Навигация | Бургер-меню | Бургер или горизонт. | Горизонтальная |
| Hero | 1 колонка, h1=4xl | 1 колонка, h1=4xl | 2 колонки, h1=5xl |
| Каталог | 1 колонка карточек | 2 колонки | 3 колонки |
| Продукт | Галерея → инфо (стек) | 2 колонки | 2 колонки |
| Таблица ТТХ | Горизонтальный скролл | Полная ширина | Полная ширина |
| Footer | Стек, аккордеоны | 2 колонки | 4 колонки |
| Секции padding | 32px | 48px | 80px |

### Touch-оптимизация

- Минимальный tap-target: 44×44px (Apple HIG).
- Кнопки: full-width на мобильном.
- Телефон кликабельный: `<a href="tel:...">`.
- Мессенджеры: иконки 48×48px с отступами.

## 11. «СИЛЬНАЯ ПОДАЧА» — КАК ДОСТИЧЬ

Собственник 2 раза использовал слово «сильно». Вот конкретные дизайн-решения:

### Крупные цифры-метрики

```
[С 2015 года]  [2017 — собственное пр-во]  [Тысячи плат/год]  [3 страны]
```
Шрифт: 6xl (60px), weight 700, цвет brand. Подпись: sm, text-secondary.

### Полноширинные фото производства

На Главной и странице Контрактного производства — фото цеха во всю ширину экрана (с overlay текстом). Демонстрация масштаба.

### Секция «Наши возможности» (Контрактное производство)

Grid из карточек с иконками: SMD-монтаж, DIP-монтаж, Программирование МК, Тестирование, Сборка, Отмывка. Каждая карточка = компетенция. Визуально: «мы умеем всё».

### Сертификаты на видном месте

Не в подвале, а в отдельной секции: логотипы стандартов (ТР ТС), номера деклараций, даты. Визуально: «мы сертифицированы».

### Профессиональная фотография продукции

Единый стиль, нейтральный фон, несколько ракурсов. Не "фото на коленке" — продуктовая съёмка.
